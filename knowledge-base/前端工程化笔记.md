## 一、Nginx 配置

### 1.1 常用命令

```bash
# 启动/停止/重载
nginx                    # 启动
nginx -s stop           # 快速停止
nginx -s quit           # 优雅停止
nginx -s reload         # 重载配置（不中断服务）

# 检查配置
nginx -t                # 测试配置语法
nginx -T                # 查看完整配置

# 查看状态
nginx -v                # 版本
ps aux | grep nginx     # 进程状态
```

### 1.2 location 匹配优先级

```
= /exact/match    → 精确匹配（最高优先级）
^~ /prefix        → 前缀匹配，不含正则
~ 正则           → 区分大小写的正则匹配（按顺序）
~* 正则          → 不区分大小写的正则匹配
/                → 通用匹配（最低优先级）
```

### 1.3 静态资源缓存配置

```nginx
# 强缓存 + 协商缓存 组合策略
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;                    # 强缓存 30 天
    add_header Cache-Control "public, immutable";
    access_log off;
}

# HTML 文件不缓存（确保内容及时更新）
location ~* \.html?$ {
    expires -1;
    add_header Cache-Control "no-cache, must-revalidate";
}
```

### 1.4 PC 端代理请求配置

```nginx
location /api/ {
    proxy_pass http://backend-server;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    
    # 超时设置
    proxy_connect_timeout 30s;
    proxy_read_timeout 60s;
}
```

### 1.5 gzip 压缩

```nginx
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;          # 压缩级别 1-9，6 是性价比最高的
gzip_min_length 1000;       # 小于 1KB 的文件不压缩
gzip_types text/plain text/css application/json application/javascript text/xml;
```

### 1.6 try_files 指令

```nginx
# 单页应用（SPA）路由配置
location / {
    try_files $uri $uri/ /index.html;
}

# 执行顺序：
# 1. 尝试匹配 $uri（精确文件）
# 2. 匹配 $uri/（目录）
# 3. 都不匹配则回退到 /index.html
```

### 1.7 rewrite 规则

```nginx
# 路径重写
rewrite ^/gw/pjmgApi/api/(.*)$ /api/$1 break;

# 移动端跳转
if ($http_user_agent ~* "(mobile|android|iphone)") {
    rewrite ^(.*)$ /mobile/$1 break;
}
```

### 1.8 移动端与 PC 端隔离发布

```nginx
# 方案：通过 user-agent 判断
upstream pc_backend { server 192.168.1.10; }
upstream mobile_backend { server 192.168.1.11; }

server {
    listen 80;
    
    # PC 端
    location / {
        proxy_pass http://pc_backend;
    }
    
    # 移动端
    location /mobile/ {
        proxy_pass http://mobile_backend;
    }
}
```

### 1.9 日志字段解析

```nginx
log_format main '$remote_addr - $remote_user [$time_local] '
                '"$request" $status $body_bytes_sent '
                '"$http_referer" "$http_user_agent" '

```

```
   → echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf

3. 插件兼容性问题
```

```javascript
// 大数据量分批渲染
function batchRender(items, batchSize = 50) {
  let index = 0
  
  function processBatch() {
    const batch = items.slice(index, index + batchSize)
    for (const item of batch) {
      renderItem(item)
    }
    index += batchSize
    
    if (index < items.length) {
      requestAnimationFrame(processBatch)
    }
  }
  
  requestAnimationFrame(processBatch)
}
```

## 七、动态定义表格操作列

### 7.1 可配置操作列的表格组件

基于 Vue + Element UI 实现灵活可配置的操作列，让操作按钮能动态注入。

```javascript
// 操作列配置项
const tableColumnList = [
  {
    label: '序号',
    field: 'number',
    colMinW: 54,
    fixed: true
  },
  {
    label: '姓名',
    field: 'member',
    colMinW: 150
  },
  {
    label: '类型',
    field: 'memberTypeName',
    colMinW: 100
  }
]

// 动态操作列配置
const operationConfig = [
  { label: '编辑', type: 'primary', action: 'edit', show: row => !row.disabled },
  { label: '删除', type: 'danger', action: 'delete', show: row => row.canDelete },
  { label: '查看', type: 'default', action: 'view' }
]
```

### 7.2 核心实现思路

```vue
<script setup>
const props = defineProps({
  columnConfig: Array,   // 表格列定义
  operationConfig: Array, // 操作按钮配置
  data: Array             // 表格数据
})

const emit = defineEmits(['action'])

// 根据操作配置动态渲染按钮
function getButtonProps(action, row) {
  const cfg = props.operationConfig.find(c => c.action === action)
  if (!cfg) return {}
  // 返回按钮配置（可覆盖默认）
  return {
    type: cfg.type || 'default',
    disabled: cfg.disabled?.(row) || false
  }
}
</script>
```

### 7.3 关键设计原则

- **配置与渲染分离**：操作列定义为纯配置，不耦合业务逻辑
- **按行控制显隐**：通过 `show` 函数支持按行数据控制按钮显隐
- **统一事件处理**：所有操作按钮触发统一 `action` 事件，父组件处理具体逻辑

### 8.3 日期范围计算工具函数

```javascript
/**
 * 获取当前月份向前 N 个月的年份-月份字符串数组
 * @param {number} months 向前几个月
 * @param {string} format 输出格式 'YYYY-MM' 或 'YYYY年MM月'
 * @returns {string[]}
 */
function getPastMonths(months = 12, format = 'YYYY-MM') {
  const result = []
  const now = new Date()
  
  for (let i = 0; i < months; i++) {
    const d = new Date(now.getFullYear(), now.getMonth() - i, 1)
    const year = d.getFullYear()
    const month = String(d.getMonth() + 1).padStart(2, '0')
    result.push(format === 'YYYY-MM' ? `${year}-${month}` : `${year}年${month}月`)
  }
  return result
}

/**
 * 计算近一个月的日期范围 [start, end]
 */
function getLastMonth() {
  const end = new Date()
  const start = new Date(end.getFullYear(), end.getMonth() - 1, end.getDate())
  return [start.toISOString().slice(0, 10), end.toISOString().slice(0, 10)]
}

/**
 * 计算近一年的日期范围 [start, end]
 */
function getLastYear() {
  const end = new Date()
  const start = new Date(end.getFullYear() - 1, end.getMonth(), end.getDate())
  return [start.toISOString().slice(0, 10), end.toISOString().slice(0, 10)]
}
```
